## 1. 架構總覽

```
v13.0.2 架構: Event Bus + State Manager + 純函數計算層
```

### 核心設計原則

| 原則 | 說明 |
|------|------|
| **零直接調用** | 模組間通過 Events.emit()/on() 通信，不直接互相調用 |
| **單一數據源** | 所有數據通過 State.get()/set() 讀寫 |
| **純函數計算** | calc/ 層無副作用，100% 可測試 |
| **Store 統一入口** | 所有 localStorage 讀寫通過 Store 模組 |
| **橋接層隔離** | HTML onclick → 全局函數必須在 bridge.js 提供 |
| **墓碑機制** | Firebase 刪除寫墓碑對象，客戶端側過濾 |
| **mergeTxs 時間戳決勝** | 衝突時取 `_updatedAt` 大的，遠端勝出 |

### 數據流方向

```
用戶操作 → data/ CRUD 函數 → State.update() → Store.save()
                                    ↓
                              Events.emit()
                                    ↓
                    ┌───────────────┼───────────────┐
                    ↓               ↓               ↓
            app.js 監聽器    bridge.js 監聽器   頁面渲染
            (重渲染頁面)    (更新下拉列表)   (render*)
                                    ↓
                    sync/*ToFirebase() → Firebase
```

### 同步流方向

```
┌──────────────┐    即時推送     ┌───────────┐    監聽拉取    ┌──────────────┐
│  手機 (Device A) │ ──syncTxToFB──→ │ Firebase  │ ←──watcher── │  電腦 (Device B) │
│  CRUD → State   │                │  RTDB     │              │  mergeTxs → State │
│       → Store   │ ←──watcher──  │           │ ──syncTxToFB──→ │       → Store    │
└──────────────┘    監聽拉取     └───────────┘    即時推送     └──────────────┘
```

## 4. Event Bus events.js

```javascript
var Events = {
  on:       function(event, handler)  // 訂閱，返回取消函數
  off:      function(event, handler)  // 取消訂閱
  emit:     function(event, ...args)  // 發布 (handler 錯誤不影響其他)
  once:     function(event, handler)  // 一次性訂閱
  debug:    function(enabled)         // 開啟/關閉調試
  listAll:  function()                // 已註冊事件列表 { event: count }
  reset:    function()                // 清除所有監聽器
};
```

特點：
- 每個 handler 獨立 try-catch，一個失敗不影響其他
- emit 自動複製 arguments (去掉 event 名)
- on() 返回取消訂閱函數

**監聽器清單** (startWatchers):

|---|------|---------|---------|

**★ 代理名單智能監聽策略 (AGENT_LIST)**:
```

  例外: 本地為空 + 遠端有數據 → 初始化接受遠端

後續觸發:
  CASE 1: remote=[] && local>0   → 刪除同步 (清空本地) ★
  CASE 2: local=[] && remote>0   → 新增同步 (接受遠端)
  CASE 3: 兩邊都有但內容不同    → remote 權威覆蓋 (Firebase 為最終共識) ★

★ 絕對不用取并集 (MERGE) — 取并集會導致被刪代理復活！
```

## 19. 同步邏輯全貌

### 墓碑機制

```
刪除操作:
  removeTxFromFirebase(fbKey)
    → set({ _fbKey, _deleted:true, _updatedAt: Date.now() })
    → Firebase 中保留墓碑對象

mergeTxs (merger.js):
  合併 local + remote → 保留墓碑 (_deleted:true)
  ★ 不過濾墓碑 — 墓碑需要寫回 Firebase 供其他設備接收

watchers.js (客戶端):
  mergeTxs 結果 → filter(r => !r._deleted)
  ★ State 中不含墓碑，墓碑僅存在於 Firebase

uploader.js (批量推送):
  transaction() → mergeTxs(local, remote)
  ★ 寫回 Firebase 時含墓碑，確保其他設備能收到刪除通知
```

### 代理名單同步 (⭐ 最終正確策略)

```
工作原理:
  Agent A 刪除 "999":
    1. removeAgent("999") → State agentList = ["111","888"]
    2. syncAgentListToFirebase(["111","888"]) → Firebase agentList = ["111","888"]
    3. Events.emit(AGENT_LIST_UPDATED, ["111","888"])

  Agent B 監聽器接收:
    1. watchers → onValue → remote = ["111","888"], local = ["111","888","999"]
    2. sortedCompare → localSorted ≠ remoteSorted
    3. CASE 3: remote 權威覆蓋 → State = ["111","888"]
    4. Events.emit(AGENT_LIST_UPDATED) → UI 更新

關鍵規則 (6 行代碼的本質):
  1. 首次觸發跳過 (避免頁面載入時被覆蓋)
  2. 相同內容跳過
  3. remote=[] → 清空本地 (刪除同步)
  4. local=[] → 接受遠端 (新增同步)
  5. 兩邊有數據 → remote 覆蓋 (Firebase 權威來源)
  6. ★ 絕不用取并集 — 取并集 = 被刪代理復活
```

### syncUploadAll 中 agentList 的 IIFE 模式

```javascript
// ★ 關鍵修復: 不在函數入口捕獲 agentList
//    改為在 set() 回調內實時讀取 State
(function() {
    var _al = State.get('agentList');
    db.ref(FB_PATH.AGENT_LIST).set(_al.slice().sort(), callback);
})();
// 這樣即使 watchers 在 syncUploadAll 執行中途更新了 State，
// set() 時讀到的是最新值，不會推送舊數據覆蓋 Firebase
```

### syncAgentListToFirebase 的 enqueue 模式

```javascript
// ★ 關鍵修復: enqueueUpload 閉包不捕獲參數
enqueueUpload(function() {
    syncAgentListToFirebase(State.get('agentList'));  // 執行時實時讀取
});
// 而非:
enqueueUpload(function() { syncAgentListToFirebase(agentList); });  // ← 捕獲舊值！
```

## 22. 代理同步歷史教訓

歷經 **14 輪修復** 後的關鍵經驗：

### 最終的正確策略

1. **Firebase 權威覆蓋** — 代理名單同步使用 `remote 覆蓋 local`，而非取并集
2. **IIFE 實時讀取** — syncUploadAll 在 set() 回調內讀取 State，不捕獲閉包舊值
3. **enqueue 實時讀取** — syncAgentListToFirebase 的 enqueue 閉包使用 `State.get()`
4. **watchers 智能策略** — 區分首次/刪除/新增/覆蓋四種場景
5. **syncDownloadAll 一致性** — 與 watchers 的 CASE 1-3 邏輯完全一致

### 已修復的致命缺陷

| # | 缺陷 | 修復 |
|---|------|------|
| 1 | syncUploadAll 捕獲舊變數 → 推送舊數據 | IIFE + 實時 State.get() |
| 2 | enqueueUpload 閉包捕獲舊參數 | 改用 State.get('agentList') |
| 3 | watchers CASE 3 取并集 → 被刪代理復活 | 改為 remote 權威覆蓋 |
| 4 | syncDownloadAll CASE: local>0 && remote=[] 只 return 不清空 | 改為清空本地 |
| 5 | Push-only 策略 (無監聽器) → 永遠不知道被刪除 | 恢復智能監聽器 |

### 同步系統不變規則

1. **永遠修改 src/ 或 css/**，不直接改 dist/ 或 docs/
2. **修改後必須 `node build.js`**（測試不通過不允許部署）
3. **HTML onclick 必須在 bridge.js 提供對應全局函數**
4. **try-catch 必須有 console.error 輸出**（不可靜默吞錯）
5. **硬編碼顏色通過 UI_COLORS 常量引用**
6. **mergeTxs/mergeWallets/mergeBookings 不過濾墓碑** — 由 watchers.js 在客戶端側過濾
7. **agentList 同步絕不用取并集** — 取并集 = 刪除無效
8. **所有 sync*ToFirebase 函數在 _db 為 null 時入隊** — 不丟失寫入
